# 文明摇篮 / Cradles Of Civilization

一个受“三颗恒星文明循环”启发的原创文字策略小游戏。

原创企划 / Original concept: Noah Walker.

当前版本：`v0.3`

## 运行

直接用浏览器打开 `index.html`。

## 打包分享

运行 `npm run package` 会生成 `dist/cradles-of-civilization/` 和 `dist/cradles-of-civilization.zip`。把 zip 发给朋友即可；对方解压后双击 `index.html` 就能玩。存档保存在各自浏览器本机，不会跟着 zip 传播。

## Windows 桌面版

商业发布路线使用 Tauri。当前工程已经有 Windows 壳、安装包配置、Rust 后端入口和 GitHub Actions 构建流程。

- 本地生成桌面前端资源：`npm run desktop:build-ui`
- 本机有完整 Tauri/Rust 环境时构建：`npm run desktop:windows`
- 在 GitHub 上构建 Windows 安装包：进入 Actions，手动运行 `Build Windows Desktop`，完成后下载 `cradles-of-civilization-windows` artifact。

注意：当前版本先完成桌面封装和后端入口，游戏规则主体仍在前端。收费发布前应继续把核心结算、事件和平衡数值迁入 `src-tauri/src/engine.rs`，并做版权引用审计。

## 基础规则

- `SC` 是科学，`BE` 是神学，二者上限都是 `20000`。
- `SC/BE` 各自有趋势值，代表下一年会自然兑现的知识增产量；趋势由上一年趋势、经济、人口、秩序、路线压制、行动和事件共同决定。
- `POP` 是人口，没有硬上限，但人口过高会引发额外风险。自然人口变化围绕文明承载力形成 S 型曲线，科技、神学、经济和秩序只提供小幅修正。
- `ECO` 是经济资本，会被事件、行动成本和部分彩蛋影响；低位经济会出现少量社会自救缓冲，但财政仍可能归零。
- `LA` 是文学艺术，最高为 `20000`；它主要作为 EERF 的线性保存增幅，满值时可让灾后保存达到毁灭轮 `SC/BE` 的约一半、正向趋势的八成。
- `POP/ECO/LA/秩序` 也会显示趋势值，但这些趋势只记录最近一年真实变化，不额外改变底层演化模型。
- v0.3 加入第一版地图系统：九个区域会显示本国、中立与敌国归属，地图会表达扩张、失守、征服和灭国。
- 军事系统包含军力、进攻、防守、边境攻防、灭国判断和全图征服判断；K/L 结局从 v0.3 起进入前台判定。
- 秩序按 `20` 分级：`0-20` 无政府、`20-40` 封建、`40-60` 君主立宪、`60-80` 资本主义、`80-100` 极权国家。
- `EERF` 是极端环境抵抗设施，建造和升级会消耗经济、科学、神学和人口，但能提高下一代文明初始人口，并保存下一代的 `SC/BE` 与正向趋势。
- EERF 第 `2/3/4/5` 级分别需要 `SC 2000/4000/8000/16000` 解锁。
- 新世界默认使用当前时间生成种子；也可以在规则栏或结算页手动输入数字种子后开始。访问 `index.html?seed=1058` 也会直接用指定种子开局。每一年会用该种子延伸出的伪随机序列掷出 `Rand`。
- 同一年还会后台掷出隐藏 `SPEC`，范围为 `1-5000`，只在命中特殊事件时执行；特殊事件栏常驻显示，未命中特殊事件时显示“日光之下，并无新事。”
- 命中特殊事件时，屏幕会显示事件名、`SPEC` 数字、说明和本次变化。
- 普通随机事件和状态事件里的 `SC`/`BE` 基础单项变化被压到 `50` 以内；自然知识增长由趋势兑现，特殊事件和玩家行动可以超过。
- 行动收益被压低，文明需要长期积累才会跨越时代。
- 行动后会结算系统压力：SC/BE 会互相压制，接近均衡时压制减弱；POP 会按承载力 S 型增长或回落，ECO 使用简化的 Solow-Cobb-Douglas 经济模型结算，随后生成下一年的 SC/BE 趋势。
- 秩序作为组织 Buff 参与系统压力：神学会抬高秩序目标，秩序会提高科学年度增长，科学发展继续反向压制神学。
- 经济模型中，`ECO` 近似资本存量，`POP` 近似劳动力，二者共同产生年度产出并呈边际递减；`SC` 提高全要素生产率，`BE` 作为教义摩擦压低可投资产出。年度 `ECO` 变化约等于生产性投资减去折旧、人口维护、知识行政、EERF 维护与过载成本。
- `ECO` 为 `0` 时进入经济危机，除“重启财政”外的行动会锁死，所有正向 `SC`/`BE` 发展都会被冻结。
- “增强经济”可以提高 `ECO`，但会牺牲一部分人口、科学和神学推进。
- EERF 有 `0-5` 级，灾变时会保存火种人口、少量知识和极少量正向趋势，并在灾变后降级。高等级保存更多知识，`BE` 保存倍率约为同级 `SC` 的 `1.08` 倍。
- 建造或升级 EERF 需要保留最低可持续人口；如果人口成本会把本代文明扣穿底线，行动会被锁住。
- 如果建造或升级 EERF 的同一年掷中硬灾变，工程会先完成封门，再结算文明毁灭与火种保存。
- 世界事件和隐藏特殊事件先结算；如果文明没有毁灭，玩家选择的行动再结算。普通随机事件和普通隐藏事件不会把人口扣穿最低可持续人口，明确标记为穿透的事件除外。
- 部分 `Rand` 会直接触发毁灭一代文明的灾变，旧文明会等待玩家点击“重启文明”，再按 EERF 火种重新启动。
- 毁灭不是游戏终点。多数 `A-L` 结局条件满足后只会点亮“脱离苦海”，玩家可以继续发展，也可以手动结算；候选终局只属于当前这代文明，本代毁灭后即失效。国家灭亡会自动进入 `L`，全图征服会点亮 `K`。
- 自动结算只保留少数情况：单线 `SC` 抵达上限且 `BE` 未同步封顶时会自动结算 `A`，单线 `BE` 抵达上限且 `SC` 未同步封顶时会自动结算 `B`；双线同时封顶会转入双相判断。隐藏 `C` 只会在连续 `18` 代文明毁灭且最高 `SC` 都低于 `1600` 时自动结算；当低科学连败接近 `C` 时，普通废墟结局 `G` 会让位给隐藏判定。`I` 会在连续 `7` 代文明以无政府秩序收束时自动结算；`J` 会在连续 `4` 代文明都曾把 `LA` 推至满值时自动结算。
- “重启财政”和“重启文明”是两个不同选项：经济为 `0` 时重启财政，文明毁灭时重启文明。

## 终局统计

游戏不再提供跨局数值增益。每次抵达终局只会记录结局种类、总达成次数和最近一次终局，作为后续成就系统的基础。开始新世界不会清空终局统计；尚未触发过的结局会隐藏名称，显示为 `B-???` 这类格式。

## 时代

科学时代按 `SC` 划分：石器、铜石并用、青铜、铁器、古典机械、蒸汽、电气、原子、信息、太空、星际航行、宇宙工程、宇宙灯塔。

神学时代按 `BE` 划分：巫祝萌芽、图腾祭司、祖灵城邦、神权律法、经院神学、圣城体系、星火教会、宇宙神学、沉默圣殿、终末预言、唯一真言、归零信标、神圣闭环。

## 文明档案

每次文明毁灭时会封存一份档案，记录存续年数、峰值 `SC/BE/LA/POP/ECO`、灭亡原因和本代的重要特殊事件。

## 快捷键

每个按钮都会在界面上显示自己的快捷键。当前映射为：`S` 科学，`B` 神学，`P` 人口，`Shift+B` 均衡治理，`Z` 维持秩序，`1` 打压神学，`2` 打压科学，`H` 脱水，`L` 文学艺术，`E` 经济，`M` 发动远征，`V` 征兵，`X` 边疆戒严，`C` 强化王权，`F` 建造 EERF，`U` 升级 EERF，`O` 重启财政，`R` 重启文明，`T` 脱离苦海，`Shift+L` 清空编年史，`Shift+N` 新世界。

## 操作辅助

按钮锁定时会直接显示原因。主界面显示数值卡、卡内趋势、战略地图和军事状态；折线图不再占用主界面，而是在结局页通过“查看文明数据”打开。规则栏保持短版摘要，终局观测、终局统计、EERF 保存预估和文明档案仍在右侧。结局页会显示本局复盘，并可复制带种子的挑战链接。

## 结局

终局条件会检查 `SC`、`BE`、`LA`、`POP`、`ECO`、秩序、地图、军力、文明代际次数与毁灭情境。科学封顶、神学封顶、科学撤离、神学撤离、双相平衡、循环废墟、低智停滞、技术威权、罗马化崩盘、永志不忘、全图征服和国家灭亡都可能成为最终结局。除少数自动结算条件外，玩家需要点击“脱离苦海”才会真正进入结局页。

## 后台预留

v0.3 起特殊政策系统进入前台，首批包括征兵、边疆戒严和强化王权；政策拥有门槛、冷却、经济/人口/秩序代价和军事效果。

## 扩展事件

随机事件主要在 `game.js` 的 `specialEventFor`、`doomEvent`、`contextualEventFor` 和 `baseEvent` 中。

趋势升级或降级只会显示在特殊播报栏，不会写进编年史。

想加新的隐藏彩蛋，优先改 `specialEventFor(spec, rng)`。命中特殊事件时，`SPEC` 会显示在播报标题里。
