# 文明摇篮 / Cradles Of Civilization

一个受“三颗恒星文明循环”启发的原创文字策略小游戏。

## 运行

直接用浏览器打开 `index.html`。

## 基础规则

- `SC` 是科学，`BE` 是神学，二者上限都是 `20000`。
- `POP` 是人口，没有硬上限，但人口过高会引发额外风险。
- `ECO` 是经济，会被事件、行动成本和部分彩蛋影响。
- `EERF` 是极端环境抵抗设施，建造和升级会消耗经济、科学、神学和人口，但能提高下一轮文明初始人口，并少量保存下一轮的 `SC/BE`。
- 每一轮先用时间种子生成的伪随机序列掷出 `Rand`。
- 同一轮还会后台掷出隐藏 `spec`，范围为 `1-5000`，不显示，只在命中特殊事件时执行。
- 命中特殊事件时，屏幕会显示事件名、说明和本次变化，但仍不显示 `spec` 数字。
- 普通随机事件和自然漂移里的 `SC`/`BE` 单项变化被压到 `50` 以内；特殊事件和玩家行动可以超过。
- 行动收益被压低，文明需要长期积累才会跨越时代。
- 行动后会结算系统压力：SC/BE 会互相压制，接近均衡时压制减弱；POP 和 ECO 会受承载力、秩序、生产力与贫困压力共同影响。
- `ECO` 为 `0` 时进入经济危机，除“重启财政”外的行动会锁死，所有正向 `SC`/`BE` 发展都会被冻结。
- “增强经济”可以提高 `ECO`，但会牺牲一部分人口、科学和神学推进。
- EERF 有 `0-5` 级，灾变时会保存火种人口和少量知识，并在灾变后降级。
- 世界事件和隐藏特殊事件先结算；如果文明没有毁灭，玩家选择的行动再结算。
- 部分 `Rand` 会直接触发毁灭一轮文明的灾变，旧文明会等待玩家点击“重启文明”，再按 EERF 火种重新启动。
- 毁灭不是游戏终点。多数 `A-H` 结局条件满足后只会点亮“结算终局”，玩家可以继续发展，也可以手动结算。
- 自动结算只保留少数情况：`SC/BE` 抵达上限会自动结算 `A/B`；隐藏 `C` 只会在连续 `3` 次文明毁灭时都满足信仰断代条件后自动结算。
- “重启财政”和“重启文明”是两个不同选项：经济为 `0` 时重启财政，文明毁灭时重启文明。

## 时代

科学时代按 `SC` 划分：石器、铜石并用、青铜、铁器、古典机械、蒸汽、电气、原子、信息、太空、星际航行、宇宙工程、宇宙灯塔。

神学时代按 `BE` 划分：巫祝萌芽、图腾祭司、祖灵城邦、神权律法、经院神学、圣城体系、星火教会、宇宙神学、沉默圣殿、终末预言、唯一真言、归零信标、神圣闭环。

## 文明档案

每次文明毁灭时会封存一份档案，记录存续轮数、峰值 `SC/BE/POP/ECO`、灭亡原因和本代的重要特殊事件。

## 结局

终局条件会检查 `SC`、`BE`、`POP`、`ECO`、秩序、文明轮回次数与毁灭情境。科学撤离、神学撤离、科学主导改革、神学主导改革、双相平衡、技术威权、信仰断代和循环废墟都可能成为最终结局。除少数自动结算条件外，玩家需要点击“结算终局”才会真正进入结局页。

## 扩展事件

随机事件主要在 `game.js` 的 `specialEventFor`、`doomEvent` 和 `baseEvent` 中。

想加新的隐藏彩蛋，优先改 `specialEventFor(spec, rng)`。这里的 `spec` 不会显示给玩家。
