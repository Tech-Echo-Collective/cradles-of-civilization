# 文明摇篮 / Cradles Of Civilization

一个受“三颗恒星文明循环”启发的原创文字策略小游戏。

## 运行

直接用浏览器打开 `index.html`。

## 基础规则

- `SC` 是科学，`BE` 是神学，二者上限都是 `20000`。
- `POP` 是人口，没有硬上限，但人口过高会引发额外风险。自然人口变化围绕文明承载力形成 S 型曲线，科技、神学、经济和秩序只提供小幅修正。
- `ECO` 是经济资本，会被事件、行动成本和部分彩蛋影响；低位经济会出现少量社会自救缓冲，但财政仍可能归零。
- `EERF` 是极端环境抵抗设施，建造和升级会消耗经济、科学、神学和人口，但能提高下一代文明初始人口，并少量保存下一代的 `SC/BE`。
- EERF 第 `2/3/4/5` 级分别需要 `SC 2000/4000/8000/16000` 解锁。
- 新世界默认使用当前时间生成种子；也可以在规则栏或结算页手动输入数字种子后开始。访问 `index.html?seed=1058` 也会直接用指定种子开局。每一年会用该种子延伸出的伪随机序列掷出 `Rand`。
- 同一年还会后台掷出隐藏 `spec`，范围为 `1-5000`，不显示，只在命中特殊事件时执行。
- 命中特殊事件时，屏幕会显示事件名、说明和本次变化，但仍不显示 `spec` 数字。
- 普通随机事件、状态事件和自然漂移里的 `SC`/`BE` 基础单项变化被压到 `50` 以内；特殊事件、玩家行动和永久增益加成可以超过。
- 行动收益被压低，文明需要长期积累才会跨越时代。
- 行动后会结算系统压力：SC/BE 会互相压制，接近均衡时压制减弱；POP 会按承载力 S 型增长或回落，ECO 使用简化的 Solow-Cobb-Douglas 经济模型结算。
- 秩序作为组织 Buff 参与系统压力：神学会抬高秩序目标，秩序会提高科学年度增长，科学发展继续反向压制神学。
- 经济模型中，`ECO` 近似资本存量，`POP` 近似劳动力，二者共同产生年度产出并呈边际递减；`SC` 提高全要素生产率，`BE` 作为教义摩擦压低可投资产出。年度 `ECO` 变化约等于生产性投资减去折旧、人口维护、知识行政、EERF 维护与过载成本。
- `ECO` 为 `0` 时进入经济危机，除“重启财政”外的行动会锁死，所有正向 `SC`/`BE` 发展都会被冻结。
- “增强经济”可以提高 `ECO`，但会牺牲一部分人口、科学和神学推进。
- EERF 有 `0-5` 级，灾变时会保存火种人口和少量知识，并在灾变后降级。高等级保存更多知识，`BE` 保存倍率约为同级 `SC` 的 `1.08` 倍。
- 建造或升级 EERF 需要保留最低可持续人口；如果人口成本会把本代文明扣穿底线，行动会被锁住。
- 如果建造或升级 EERF 的同一年掷中硬灾变，工程会先完成封门，再结算文明毁灭与火种保存。
- 世界事件和隐藏特殊事件先结算；如果文明没有毁灭，玩家选择的行动再结算。普通随机事件和普通隐藏事件不会把人口扣穿最低可持续人口，明确标记为穿透的事件除外。
- 部分 `Rand` 会直接触发毁灭一代文明的灾变，旧文明会等待玩家点击“重启文明”，再按 EERF 火种重新启动。
- 毁灭不是游戏终点。多数 `A-H` 结局条件满足后只会点亮“结算终局”，玩家可以继续发展，也可以手动结算。
- 自动结算只保留少数情况：`SC/BE` 抵达上限会自动结算 `A/B`；隐藏 `C` 会在连续 `2` 次文明毁灭时都满足信仰断代条件，或连续 `3` 代文明重启后仍处于低知识、低发展停滞时自动结算。若一代文明最高 `SC` 与 `BE` 都不超过 `500`，它会直接计入低知识停滞。
- “重启财政”和“重启文明”是两个不同选项：经济为 `0` 时重启财政，文明毁灭时重启文明。

## 造物灵感

只有 `A` 结局会生成 `1` 点造物灵感。造物灵感保存在独立永久存档里，开始新世界不会清空。玩家可以把灵感投入科学、神学、经济或人口四条路线，每级让对应正向增长提高 `5%`，单线最高 `5` 级。

升级费用按全局总升级数计算：前 `5` 次升级每次费用 `1`，第 `6-10` 次每次费用 `2`，以此类推。也就是说，即便科学线点满，神学线也不会回到低费用。

## 时代

科学时代按 `SC` 划分：石器、铜石并用、青铜、铁器、古典机械、蒸汽、电气、原子、信息、太空、星际航行、宇宙工程、宇宙灯塔。

神学时代按 `BE` 划分：巫祝萌芽、图腾祭司、祖灵城邦、神权律法、经院神学、圣城体系、星火教会、宇宙神学、沉默圣殿、终末预言、唯一真言、归零信标、神圣闭环。

## 文明档案

每次文明毁灭时会封存一份档案，记录存续年数、峰值 `SC/BE/POP/ECO`、灭亡原因和本代的重要特殊事件。

## 快捷键

每个按钮都会在界面上显示自己的快捷键。当前映射为：`S` 科学，`B` 神学，`P` 人口，`Shift+B` 均衡治理，`Z` 维持秩序，`1` 打压神学，`2` 打压科学，`H` 冬眠，`E` 经济，`F` 建造 EERF，`U` 升级 EERF，`O` 重启财政，`R` 重启文明，`T` 结算终局，`Shift+S` 科学灵感，`Shift+Y` 神学灵感，`Shift+E` 经济灵感，`Shift+P` 人口灵感，`Shift+L` 清空编年史，`Shift+N` 新世界。

## 操作辅助

按钮锁定时会直接显示原因。规则栏会显示当前最接近的非隐藏终局，以及 EERF 本代保存预估和下级升级需求。编年史可以按全部、灾变、特殊和发展筛选；结局页会显示本局复盘，并可复制带种子的挑战链接。

## 结局

终局条件会检查 `SC`、`BE`、`POP`、`ECO`、秩序、文明代际次数与毁灭情境。科学撤离、神学撤离、科学主导改革、神学主导改革、双相平衡、技术威权、信仰断代和循环废墟都可能成为最终结局。除少数自动结算条件外，玩家需要点击“结算终局”才会真正进入结局页。

## 扩展事件

随机事件主要在 `game.js` 的 `specialEventFor`、`doomEvent`、`contextualEventFor` 和 `baseEvent` 中。

想加新的隐藏彩蛋，优先改 `specialEventFor(spec, rng)`。这里的 `spec` 不会显示给玩家。
